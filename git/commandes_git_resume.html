Résumé des commandes git

Commande Description
git status Afficher l'état du repository
Exemple 
x
$ git status
On branch master
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git restore <file>..." to discard changes in working directory)
        modified:   main.cpp
no changes added to commit (use "git add" and/or "git commit -a")

 
git init Initialise un nouveau repository
Exemple 
x
$ git init
Initialized empty Git repository in C:/Repositories/Cours/lab/repo/.git/
$ ls -a
./  ../  .git/

 
git clone <adresse_repo_distant> Clone un repository distant à l'adresse <adresse_repo_distant> dans le dossier courant
Exemple 
x
$ git clone https://github.com/chtimy/Cours.git
Cloning into 'Cours'...
remote: Enumerating objects: 130, done.
remote: Counting objects: 100% (130/130), done.
remote: Compressing objects: 100% (97/97), done.
Receivingremote: Total 130 (delta 22), reused 130 (delta 22), pack-reused 0
Receiving objects: 100% (130/130), 13.38 MiB | 4.11 MiB/s, done.
Resolving deltas: 100% (22/22), done.

 
git add <path> Ajoute un fichier/dossier à la liste d'enregistrement pour le prochain commit. Si un dossier est donné, tous les fichiers sont ajoutés récursivement (exceptés ceux dans le fichier .gitignore).
Exemple 
x
$ git status
On branch master
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git restore <file>..." to discard changes in working directory)
        modified:   main.cpp
no changes added to commit (use "git add" and/or "git commit -a")
$ git add .
$ git status
On branch master
Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        modified:   main.cpp

 
git commit -m"<message>" Prend la "photo" de tous les fichiers ajoutés sur la liste d'enregistrement (voir commande git add <path>). "<message>" correspond au message associé au commit afin de donner plus d'information sur le contenu du commit("photo").
Exemple 
x
$ git status
On branch master
Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        modified:   main.cpp
$ git commit -m"main.cpp a change"
[master 54cb706] main.cpp a change
 1 file changed, 1 insertion(+)
$ git status
On branch master
nothing to commit, working tree clean

 
git rm --cached <file> Enlève les modifications apportées à un fichier de la liste d'enregistrement du prochain commit. Le fichier passe de l'état "staged" à l'état "unstaged".
Exemple 
x
$ git status
On branch master
Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        modified:   main.cpp
$ git restore --staged main.cpp
$ git status
On branch master
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git restore <file>..." to discard changes in working directory)
        modified:   main.cpp
no changes added to commit (use "git add" and/or "git commit -a")

 
git diff <commit-hash> Fais la différence entre les changements actuels du repository avec le commit spécifié dans <commit-hash>. Le terme HEAD correspond au dernier commit en date. Si un '^' est ajouté derrière (HEAD^), c'est le commit précédent et ainsi de suite en ajoutant d'autres '^'. Pour trouver le nom (hash) des commits, voir la commande git log.
Exemple 
x
$ git diff
diff --git a/main.cpp b/main.cpp
index 267d77b..6b6bcf3 100644
--- a/main.cpp
+++ b/main.cpp
@@ -3,5 +3,6 @@
 int main()
 {
        std::cout << "Hello world!" << std::endl;
+       std::cout << "Hello toi!" << std::endl;
        return 0;
 }
